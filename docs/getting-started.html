Importing KP health articles into AEM Edge Delivery Services

482 / 486 imported Edge Delivery Services Adobe Spectrum

This guide walks through importing a health-article page from healthy.kaiserpermanente.org into this Edge Delivery Services project. Articles are 100% default content — no blocks needed. Follow the steps in order.

Before you start

Get a DA token from a logged-in da.live session and add it to .env as DA_IMS_TOKEN=….

Heads up: the token is short-lived (~24h). Refresh it before any re-run or your uploads will fail with auth errors.

The golden rule: two HTML formats

The #1 cause of broken imports is uploading the wrong format to DA.

Rendered .plain.htmlDA authoring source
Used forlocal preview / aem.pageupload to admin.da.live
A block is…<div class="block-name">a <table> naming the block
Always convert rendered → table format before uploading. upload-verify.js does this automatically (and read-back verifies).

Steps

  1. Run the import

    Use the page-import skill (one page at a time):

    /edge-delivery-services:page-import \
  https://healthy.kaiserpermanente.org/northern-california/health-wellness/healtharticle.{slug}

  2. Generate the .plain.html

    Two content sections + a metadata section. Section 1 = article (style: tags,share), Section 2 = footnotes (style: footnotes), Section 3 = metadata block (template: article, then tags/title/description).

    Use the original KP CDN URL for the hero image. Don't use the hero block — the image sits inline.

  3. Exclude the chrome

    Skip the <header> nav, Back/Print controls, the share widget, the <footer>, and cookie banners.

  4. Write both output paths

    Local: …/healtharticle-{slug}.plain.html · DA: …/healtharticle-{slug}.html. Slug transform: replace the . separator with - and lowercase.

  5. Convert, upload, verify

    node import/upload-verify.js

    Converts blocks to DA tables, uploads via raw PUT, reads each file back, and retries up to 4×. Never use a multipart POST — DA flattens it.

  6. Preview & confirm

    curl -X POST -H "Authorization: Bearer $DA_IMS_TOKEN" \
  "https://admin.hlx.page/preview/AdobeDrago/kp-hw/main/northern-california/health-wellness/healtharticle-{slug}"

    A correct round-trip renders <div class="tags share"> and <div class="footnotes">, with title/description in <head>.
